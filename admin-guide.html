📝 如何处理订单与退款

适用于：客户来电查询订单、申请退款时的处理流程。

📍 操作路径：用户管理 → 搜索用户（手机号） → 用户详情 → 查看资产 / 查看订单 → 处理退款 / 客服快速发卡 / 延期

Step 1：用手机号搜索用户

进入「用户管理」，输入用户手机号搜索目标用户。优先使用手机号可精准定位用户身份，避免昵称重复导致的误操作。

Step 2：进入用户详情 → 查看资产

点击用户进入详情页，切换到「资产管理」标签，查看该用户持有的次卡、会籍卡、优惠券及储值余额情况。

  • 核对资产有效期、剩余次数/金额是否正常
  • 如用户反馈「找不到卡」，先在此确认是否过期、已用完或冻结

Step 3：查看订单记录

切换到「订单记录」标签，找到需要处理的订单，点击进入订单详情。

  • 确认订单状态、实付金额、支付方式
  • 如需退款，进入订单详情后切换到「预约记录」标签，查看该订单下的所有预约记录
涉及资金

Step 4：处理退款

进入订单详情页后，不同资产类型的退款入口不同，请按实际商品类型选择：

方式一：次卡 / 会籍卡退款

路径：订单详情 → 订单商品明细 → 次卡/会籍卡退款

  1. 在「订单商品明细」区域找到次卡/会籍卡商品
  2. 点击「次卡退款」或「会籍卡退款」入口
  3. 查看退款明细（金额、支付方式、原因），确认无误后提交

方式二：课程退款

路径：订单详情 → 订单商品明细 → 预约记录 → 处理预约 → 退款

  1. 在「订单商品明细」区域切换到「预约记录」标签
  2. 找到需要退款的课程记录，点击「处理预约」→「退款」
  3. 查看退款明细（金额、支付方式、原因），确认无误后提交
注意：涉及次卡、会籍卡等资产退款，务必核对用户资产余额与退款金额是否匹配。退款前建议截图留存当前资产状态，避免纠纷。款项原路退回（微信退微信、余额退余额、次卡按规则返还）。
图 2-1-1：课程退款 — 预约记录 → 处理预约 → 退款
图 2-1-2：次卡/会籍卡退款 — 订单商品明细退款入口
图中关键操作点：① 课程类需在「预约记录」标签中找到记录后，点击「处理预约」→「退款」；② 次卡/会籍卡在「订单商品明细」直接点击对应退款入口；③ 确认退款明细后提交，款项原路退回。

Step 5：用户补偿与延期

对于需要补偿的用户，在用户详情页选择「客服快速发卡」，选择卡券类型和数量提交，即时到账。

对于需要延期的次卡/会籍卡，在用户卡券详情页执行「延期」或「冻结/解冻」操作，需填写原因，所有操作均留操作日志。

注意：为防止发错，客服快速发卡仅支持固定范围内的模板，不支持在后台自行关联新的发卡模板。如需新增模板，请联系产品经理处理。
注意事项：已核销的课程订单原则上不支持退款；储值卡退款需扣除已消费金额及赠送金。所有退款操作均记录操作日志，不可删除。

🧾 如何处理发票申请

适用于：客户提交发票申请后，财务/运营人员进行开票或驳回。

📍 操作路径：发票管理 → 查看申请列表 → 点击对应申请处理

处理开票申请

点击「发票管理」即可看到发票申请列表，点击对应申请进行处理：

  1. 可开票：在税务系统开具电子发票 → 点击「上传发票」上传 PDF/图片 → 状态变为「已开票」
  2. 驳回：点击「驳回」→ 填写原因（如「订单已全额退款」「税号错误」）→ 客户可在小程序查看原因并重新提交

操作按钮差异：「待处理」显示「查看/上传/驳回」；「已开票」显示「查看」（可重新下载）；「已驳回」仅显示「查看」。

注意：仅「已完成」状态的订单才可开票；「已驳回」的申请需告知客户修改后重新提交。
图 3-2：发票申请详情 — 申请信息与发票文件预览
图中关键操作点：① 核对「发票抬头」和「企业税号」是否与用户提交一致；② 点击「上传发票」上传税务系统开具的 PDF/图片；③ 如信息有误，点击「驳回」并填写具体原因。
最佳实践：建议每日下班前集中处理当日发票申请，避免积压。

👤 如何查询与管理用户

适用于：客户来电/到店咨询、投诉处理、信息核实时。包含用户查询、资产管理、封禁操作。

📍 操作路径：用户管理 → 搜索用户 → 查看详情 / 编辑标签 / 添加备注 / 封禁管理

Step 1：搜索用户

进入「用户管理」，输入手机号搜索目标用户。列表中「资产信息」概览列可直接查看储值余额、次卡剩余次数、优惠券张数，无需进入详情页。

图 4-1：用户列表 — 支持多维度搜索，列表可直接查看资产概览
图中关键操作点：① 搜索框支持昵称/手机号/用户ID模糊搜索；② 「资产信息」列可直接查看储值余额、次卡剩余次数、优惠券张数；③ 勾选多用户后可点击「批量添加标签」。

Step 2：查看用户详情与资产

点击用户进入详情页，包含多个标签页：

  • 基本信息：昵称、ID、联系方式（点击眼睛图标查看未脱敏号码）、人脸识别状态
  • 资产管理：次卡、会籍卡、优惠券的持有情况与可用次数/有效期
  • 订单记录 / 预约记录 / 历史到店：消费、约课、签到历史
  • 封禁管理：账号封禁记录与操作历史
话术示例：「您好，查询到您目前持有「89元通用次卡包」1 张，总次数 1 次，剩余可用 1 次，有效期至 2026-12-31。」
图 4-2：用户详情 — 资产管理页

Step 3：管理用户标签与备注

在用户详情页点击「编辑标签」，可添加标签（如「高活跃」「投诉用户」「VIP」）。支持在列表页批量勾选用户后批量添加标签，方便后续营销触达。

添加备注可记录特殊需求（如「对花生过敏」「偏好早晨课程」），方便后续客服跟进。

谨慎操作

Step 4：封禁 / 解封用户

仅用于用户存在恶意占课、刷单、骚扰教练等严重违规行为时。

  1. 收集违规证据（订单记录、客服聊天记录、教练反馈）
  2. 在用户详情页切换到「封禁管理」，查看历史封禁记录
  3. 点击「封禁账号」→ 选择原因 → 设置时长（永久/临时）→ 确认
  4. 非永久封禁建议通过客服渠道告知用户原因及解封时间

解封：在「封禁管理」找到记录 → 点击「解封」→ 填写原因 → 确认，用户立即恢复正常。

图 4-3：用户违规处理 — 违规行为记录与证据
图 4-4：封禁管理 — 封禁/解封操作界面
图中关键操作点：① 在「封禁管理」标签页查看历史违规记录；② 点击「封禁账号」选择封禁原因与时长；③ 非永久封禁到期后系统自动解封，也可手动提前解封。
重要提醒：封禁操作不可逆地影响用户体验，执行前务必确认违规事实并截图留存证据。所有封禁/解封操作均记录操作日志。

📊 如何查看数据看板

适用于：每日晨会前、每周复盘时，快速掌握门店经营全貌。

📍 操作路径：数据概况 → 运营中心
每日必看

Step 1：运营中心

点击数据概况 → 运营中心查看，顶部数据卡展示核心指标：

  • 今日营收：当日总收入及环比变化
  • 活跃用户：当日活跃用户数及本月新增人数
  • 课程完成率：已完课数 ÷ 总排课数
  • 会员续费率：会员续费率及月度变化

下方图表可切换「7天 / 30天 / 90天」营收趋势，右侧展示门店营收排名，快速定位业绩异常门店。

提示：点击「导出报表」可导出 Excel；点击「大屏展示」可切换全屏看板，适合会议室投影。
最佳实践：建议每日 9:00 前查看运营中心，重点关注「今日待履约」和「异常待处理」模块，提前安排应对。

🏋️ 如何新建一门课程

适用于：引入新课程类型、新开门店需要独立课程体系时。完整流程：小程序分类 → 类目 → 排课条目 → 媒体模板 → 启用。

📍 操作路径：课程管理 → 团体课 → 小程序分类 → 类目管理 → 排课条目管理 → 媒体模板管理
必选

Step 1：新建小程序分类

小程序分类用于前端筛选栏展示。进入 课程管理 → 团体课 → 管理小程序分类 → 点击「新增」→ 填写分类名称、排序值 → 保存后点击「启用」。

必选

Step 2：新建课程类目

类目是创建排课条目的前提。进入 课程管理 → 团体课 → 类目管理 → 点击「新增类目」→ 填写类目名称、排序 → 关联至少一个小程序分类 → 添加标签 → 确认创建。

默认状态为「草稿」，需在列表页点击「启用」后小程序端才显示。删除前需确认无关联排课条目。

图 5-1：类目管理 — 课程分类配置

Step 3：新建排课条目

进入 课程管理 → 团体课 → 排课条目管理，点击「新建排课条目」：

  • 必填：课程名称、课程类型（常规/活动/体验/企业）、课程类目、时长、最小人数/建议容量、价格
  • 可选：课程标签、按教室配置容量、媒体模板
提示：排课条目是后续排课的基础，务必确认类目关联正确，否则小程序端无法筛选到该课程。
图 5-2：新建排课条目 — 课程基础属性配置
图中关键操作点：① 「课程类目」必须选择已启用的类目，否则小程序端无法筛选；② 「按教室配置容量」可为不同门店/教室分别设置人数上限；③ 填写完成后点击「保存」，状态为「草稿」，需手动点击「启用」。
必选

Step 4：新建媒体模板

进入 课程管理 → 媒体模板管理 → 选择管理条目 → 上传课程图片（列表封面、详情页头图）→ 上传课程视频（可选）→ 填写课程介绍 → 设置是否为默认模板。

列表采用分组折叠展示，支持「全部展开/收起」快速切换，可按模板ID、名称、状态筛选。

图 5-3：媒体模板管理 — 课程内容素材配置（分组折叠展示）

Step 5：启用排课条目

在排课条目列表页点击「启用」，状态变为「启用」后即可用于排课。如需在小程序端展示，还需完成后续排课和发布步骤。

完成检查：启用后确认 1）类目已启用；2）小程序分类已启用；3）排课条目已启用；4）媒体模板状态正常。

📅 如何进行排课

适用于：每周固定排课、临时加课、节假日调课。支持按门店、教室、教练三个维度创建多个排课视图，为每个视图命名以区分不同排课方案。发布后即可在前端展示。

📍 操作路径：课程管理 → 团体课 → 排课管理 → 列表视图 → 新建排课

排课前准备

排课前请确认以下信息已配置完成：

  • ✅ 排课条目已创建且状态为「启用」
  • ✅ 目标门店和教室已配置
  • ✅ 授课教练已录入系统且状态为「在职」
  • ✅ 教练已在「日程报备」中标记该时段为「可排课」
必读

排课与预约开放时间规则

城市负责人需在每周五前完成下一周课程排课。课程按时段分批开放预览与预约：

  • 下周一到周四的课程：本周五 15:00 开放预览，17:00 开放预约
  • 下周五到周日的课程：下周二 15:00 开放预览，17:00 开放预约

Step 1：新增门店

进入「门店管理」→「门店列表」→「新增门店」，填写核心信息：

  • 联系电话 / 营业时间：用户端展示
  • 容量 / 展示状态：新门店先选「装修中」

进入「门店管理」→「门禁管理」→「新增门禁设备」：

重要提醒：「开门」操作直接影响门店物理安全，执行前需核实请求人身份，非紧急情况建议由门店前台处理。

⚙️ 账号权限与操作日志

适用于：新员工入职开账号、权限调整、安全审计。

📍 操作路径：系统管理 → 账号管理 / 操作日志

账号与角色管理

进入 系统管理 → 账号管理：

  1. 点击「新建账号」，填写姓名、手机号、登录邮箱/用户名
  2. 选择角色：系统管理员 / 运营 / 客服 / 财务 / 门店负责人等
  3. 分配菜单权限：控制该账号能看到哪些模块
  4. 分配操作权限：控制能执行哪些操作（如普通运营无退款权限，客服主管可处理退款等）
  5. 保存后账号即时生效，用户可使用用户名+密码登录
权限设计原则：最小权限原则——只授予完成工作所必需的权限。涉及资金的操作（退款、卡券发放、薪酬修改）建议仅开放给主管级账号。

图中关键操作点：① 选择角色（系统管理员/运营/客服/财务等）控制可访问模块；② 「菜单权限」决定能看到哪些页面；③ 「操作权限」决定能执行哪些敏感操作（如退款、卡券发放）。

查看操作日志

所有重要操作均记录日志，用于数据安全审计：

  • 退款操作：退款金额、原因、操作人、执行时间
  • 卡券发放：发放对象、卡券类型、数量、操作人
  • 教练信息修改：修改字段、修改前/后值、操作人
  • 报表导出：导出时间、导出内容、操作人 IP
  • 登录记录：登录时间、IP 地址、登录设备

支持按时间、操作人、操作类型筛选查询。操作日志不可删除或修改，确保审计可追溯。

安全提醒：发现异常操作（如非工作时间大量退款、陌生 IP 登录）时，请立即冻结相关账号并联系技术负责人排查。

🛡️ 数据安全红线

所有后台操作人员必须遵守的合规底线。违规操作可能导致法律责任与品牌损失。

必背

用户数据安全

  • 手机号脱敏查看：用户完整手机号需点击眼睛图标才可查看，禁止批量截图或导出用户联系方式
  • 人脸数据保护：人脸识别数据仅用于门禁通行，禁止截图、下载或外传
  • 用户资产信息：不得向非本人透露用户的卡券余额、消费记录等敏感资产信息
  • 账号共用禁止：严禁将自己的账号密码告知他人，或使用他人账号操作系统
必背

资金操作安全

  • 双人复核：涉及资金的退款、卡券发放、薪酬修改等敏感操作，建议由另一名同事复核后再执行
  • 退款上限：退款金额不得超过用户实付金额，储值卡退款需扣除已消费及赠送金部分
  • 对账要求：每日下班前核对当日退款笔数与金额，确保与财务系统一致
  • 异常上报：发现非工作时间大量退款、陌生 IP 登录、异常批量导出，立即冻结账号并上报
必背

内容合规

  • 运营位素材审核：上线前确认图片/文案不涉及虚假宣传、违禁词、侵权内容
  • 协议更新：隐私政策、用户协议的更新必须经过法务审核，并在小程序中以弹窗告知用户
  • 课程描述：不得使用「疗效」「治愈」等医疗宣称词汇，避免广告法风险
违规后果：违反上述红线的操作，系统均留有不可篡改的日志记录。一经发现，将根据公司制度追究当事人责任；如涉及用户资金损失或法律风险，将移交法务与公安机关处理。
