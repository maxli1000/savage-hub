📊 如何查看每日经营数据

适用于：每日晨会前、每周复盘时，快速掌握门店经营全貌。

📍 操作路径：首页 → 数据概况 → 老板驾驶舱 / 运营中心
每日必看

Step 1：打开老板驾驶舱

登录后台后，默认进入老板驾驶舱页面。顶部的数据卡分别展示：

  • 今日营收：当日总收入及环比变化（绿色 ↑ 表示增长）
  • 活跃用户：当日活跃用户数及本月新增人数
  • 课程完成率：已完课数 ÷ 总排课数，及与行业平均的对比
  • 会员续费：会员续费率及月度变化

下方图表区域可切换查看「7天 / 30天 / 90天」营收趋势，右侧展示门店营收排名（按今日营收降序排列），快速定位业绩最好与最差的门店。

提示：点击「导出报表」可将数据导出为 Excel；点击「大屏展示」可切换至全屏看板模式，适合会议室投影。
图 1-1：老板驾驶舱 — 核心经营指标一览
每日必看

Step 2：切换至运营中心

在数据概况 Tab 中切换到运营中心，关注运营维度的数据：

  • 今日约课：已约课节数，下方标注「待履约 XX 节」，判断今日课程饱和度
  • 在职教练：当前在职教练人数，确认排课人力是否充足
  • 场地使用率：已排课时段占可用时段的比例，低于 60% 需考虑促销
  • 用户满意度：本月用户评分平均分，低于 4.5 需排查服务问题
  • 课程数据分析：按团体课/私教课/训练营查看排课、完成、取消、营收数据
  • 教练排行榜：本月营收 TOP5 教练，展示教练评分、完成率及营收金额，用于激励与绩效沟通
图 1-2：运营中心 — 日常运营数据监控
最佳实践：建议每日 9:00 前查看运营中心，重点关注「今日待履约」和「异常待处理」模块，提前安排应对。

📝 如何处理订单与退款

适用于：客户来电查询订单、申请退款时的处理流程。

📍 操作路径：订单管理 → 全部订单 → 搜索订单 / 处理退款

Step 1：搜索目标订单

进入「订单管理」→「全部订单」，可通过以下任一方式定位订单：

  • 输入订单号（系统内部编号，如 T26052...）
  • 输入支付订单号（第三方支付流水号）
  • 输入用户手机号查找该用户的全部订单
  • 按订单类型（团课 / 会籍卡 / 次卡 / 储值卡）、支付状态、时间范围筛选

Step 2：识别订单关键信息

在订单列表中，重点关注以下字段：

字段说明处理参考
订单类型团课 / 会籍卡 / 次卡 / 储值卡不同类型退款规则不同
商品信息购买的课程/卡券名称与数量核对客户描述是否一致
实付金额用户实际支付金额退款金额以此为上限
状态已完成 / 已取消 / 退款中「已完成」才可发起退款
支付方式现金支付 / 微信支付 / 支付宝 / 余额原路退回
图 2-1：订单管理列表页

Step 3：查看订单详情

点击订单编号进入详情页，可查看：

  • 订单信息：订单编号、订单类型、订单状态、创建时间
  • 支付信息：支付时间、支付渠道、支付流水号
  • 用户信息：用户昵称、用户ID、手机号
  • 订单金额明细：结算明细ID、抵扣类型（现金/优惠券/资产）、优惠券作用范围、资产ID、抵扣金额、抵扣数量
  • 订单商品明细：商品名称、子订单号、件数、单价、小计
  • 核销记录：课程签到/开卡/使用记录
  • 退款记录：历史退款申请及处理结果

在订单商品明细区域，可根据情况执行以下操作：

  • 订单补差：当订单金额需要调整时使用
  • 会籍卡退款：针对会籍卡类订单的专项退款入口
涉及资金

Step 4：处理退款申请

筛选「退款中」状态的订单，点击进入详情页处理：

  1. 查看退款明细（退款金额、支付方式、退款原因）
  2. 点击「同意退款」或「驳回」
  3. 同意后款项原路退回（微信退微信、余额退余额、次卡按规则返还）
  4. 驳回需填写驳回原因
🔒 资产类退款审批流
客服提交 → 客服主管审批 → 财务确认 → 执行退款

涉及次卡、会籍卡等资产退款，必须走完上述审批流，系统自动记录操作日志。

Step 5：用户补偿（客服快速发卡）

对于需要补偿的用户，可通过客服快速发卡进行补贴：

  1. 找到需要补偿的用户
  2. 选择「客服快速发卡」功能
  3. 选择卡券类型和数量，提交后即时到账
注意事项：已核销的课程订单（用户已上课）原则上不支持退款；储值卡退款需扣除已消费金额及赠送金部分。所有退款操作均记录操作日志，不可删除。

🧾 如何处理发票申请

适用于：客户提交发票申请后，财务/运营人员进行开票或驳回。

📍 操作路径：发票管理 → 开票申请 → 处理 / 上传发票

Step 1：查看待处理的发票申请

进入「发票管理」→「开票申请」，列表显示所有用户提交的申请。可按以下条件筛选：

  • 处理状态：待处理 / 已开票 / 已驳回 / 待开票
  • 订单号：输入关联订单号精确搜索
  • 全部类型：按发票类型筛选
  • 申请时间：按时间范围筛选

列表中重点关注以下字段：

字段说明
抬头类型个人 / 企业
发票抬头 / 企业税号开票信息，需仔细核对
处理状态待处理 / 已开票 / 已驳回 / 待开票
驳回原因被驳回时显示具体原因，方便二次处理
操作根据状态显示不同操作：「查看」「上传」「驳回」
注意：仅「已完成」状态的订单才可开票；「已驳回」的申请需告知客户修改后重新提交。
图 3-1：发票管理 — 发票申请处理列表

Step 2：处理开票申请

点击「处理」或列表中的「查看/上传/驳回」按钮，核对订单信息后操作：

  1. 信息无误可开票：在税务系统中开具电子发票 → 点击「上传发票」将 PDF/图片上传至系统 → 状态变为「已开票」
  2. 信息有误或不符合开票条件：点击「驳回」→ 填写驳回原因（如「订单已全额退款」「税号错误」）→ 客户可在小程序中查看驳回原因并重新提交

操作按钮差异：

  • 「待处理」状态：显示「查看」「上传」「驳回」
  • 「已开票」状态：显示「查看」（可重新下载发票）
  • 「已驳回」状态：显示「查看」
图 3-2：发票申请详情 — 申请信息与发票文件预览
最佳实践：建议每日下班前集中处理当日发票申请，避免积压。已开票的发票文件可在详情页随时下载复查。

👤 如何查询与管理用户

适用于：客户来电/到店咨询、投诉处理、信息核实时。

📍 操作路径：用户管理 → 搜索用户 → 查看详情 / 编辑标签 / 添加备注

Step 1：搜索用户

进入「用户管理」页面，可通过以下任一方式定位用户：

  • 昵称 / 手机号：支持模糊搜索
  • 会员等级：按等级筛选
  • 归属门店 / 教练：按服务归属筛选
  • 用户ID：客户提供的数字 ID

用户列表中可直接查看「资产信息」概览列，显示该用户的储值余额、次卡剩余次数、优惠券张数，无需进入详情页即可快速了解用户资产状况。

提示：如果客户不记得用户ID，可通过手机号后四位模糊搜索。
图 4-1：用户列表 — 通过多维度条件搜索，列表可直接查看资产概览

Step 2：查看用户详情

点击用户进入详情页，默认展示「基本信息」标签：

  • 用户昵称、ID、性别、年龄、生日：基础身份信息
  • 注册时间：可判断用户是新客还是老客
  • 联系方式：完整手机号（点击眼睛图标可查看未脱敏号码）
  • 人脸识别：显示「已录入」或「未录入」
  • 地址信息：客户维护的收货地址列表

用户详情页包含多个标签页：

标签页用途典型场景
基本信息身份信息、联系方式客户来电核实身份
资产管理次卡、会籍卡、优惠券客户询问「我还有几次课」
订单记录全部历史订单客户查询某笔消费
预约记录全部课程预约客户询问「我约了哪节课」
历史到店/签到到店和签到记录核实用户到店情况
封禁管理账号封禁记录排查账号异常
图 4-2：用户详情 — 基本信息页

Step 3：查看用户资产

切换到「资产管理」标签，可看到该用户持有的全部资产：

资产类型查看内容支持操作
次卡来源、账户编号、模板名称、总次数、可用次数查看使用记录、延期
会籍卡有效期、适用门店、权益内容查看详情、延期、冻结/解冻
优惠券优惠券名称、面额、有效期、使用条件查看使用记录
话术示例：「您好，查询到您目前持有「89元通用次卡包」1 张，总次数 1 次，剩余可用 1 次，有效期至 2026-12-31。」
图 4-3：用户详情 — 资产管理（次卡列表）

Step 4：管理用户标签

在用户详情页点击「编辑标签」，可添加或修改标签（如「高活跃」「投诉用户」「VIP」）。

批量添加标签：在用户列表勾选多个用户后，点击「批量添加标签」，统一打标签，方便后续营销触达。

Step 5：添加用户备注

在用户详情页可添加备注，记录特殊需求或沟通历史（如「对花生过敏」「偏好早晨课程」），方便后续客服跟进。

🚫 如何封禁/解封用户

适用于：用户存在恶意占课、刷单、骚扰教练等严重违规行为时。

📍 操作路径：用户管理 → 搜索用户 → 查看详情 → 封禁管理
谨慎操作

封禁操作流程

1

确认违规事实

收集用户违规证据（订单记录、客服聊天记录、教练反馈等），确保封禁有据可依。

2

进入用户封禁管理页

在用户详情页切换到「封禁管理」标签，查看该用户是否有历史封禁记录。

3

执行封禁操作

点击「封禁账号」→ 选择封禁原因 → 设置封禁时长（永久 / 临时）→ 确认操作。

4

通知用户（可选）

对于非永久封禁，建议通过客服渠道告知用户封禁原因及解封时间。

解封操作流程

在「封禁管理」页面找到已封禁记录，点击「解封」→ 填写解封原因 → 确认。解封后用户可立即恢复正常使用。

重要提醒：封禁操作不可逆地影响用户体验，执行前必须获得主管审批。误封可能导致客户投诉及品牌损失。所有封禁/解封操作均记录操作日志。

🏋️ 如何新建一门课程

适用于：引入新课程类型、新开门店需要独立课程体系时。完整流程：小程序分类 → 类目 → 排课条目 → 排课 → 审批发布。

📍 操作路径：课程管理 → 团体课 → 小程序分类 → 类目管理 → 排课条目管理 → 排课管理
可选

Step 1：新建小程序分类

小程序分类用于前端筛选栏展示，方便用户按分类查找课程。

  1. 进入 课程管理 → 团体课 → 管理小程序分类
  2. 点击「新增」，填写分类名称、排序值
  3. 保存后状态为「草稿」，在列表页点击「启用」后小程序端即可看到该分类
必选

Step 2：新建课程类目

类目用于课程归类，是创建排课条目的前提。

  1. 进入 课程管理 → 团体课 → 类目管理，点击「新增类目」
  2. 填写类目名称、排序（数字越小排序越靠前）
  3. 关联至少一个小程序分类（否则无法创建）
  4. 添加标签（如「燃脂」「拉伸」），点击「确认创建」，默认状态为「草稿」
  5. 在列表页点击「启用」，类目状态更新为「启用」，小程序端显示

列表支持按「类目ID」「类目名称」「状态」筛选搜索。每条类目可执行「查看」「编辑」「删除」操作，删除前需确认无关联的排课条目。

图 5-1：类目管理 — 课程分类配置

Step 3：新建排课条目

进入 课程管理 → 团体课 → 排课条目管理，点击「新建排课条目」：

字段填写建议是否必填
课程名称清晰易懂，同类型下不可重复必填
课程类型常规课 / 活动课 / 体验课 / 企业课必填
课程类目选择 Step 2 中创建的类目必填
课程标签如「燃脂」「拉伸」「单车」等，用于前端筛选展示可选
时长单节课时长（分钟），如 60 分钟必填
最小人数 / 建议容量如「最少1人 / 建议5人」必填
价格课程单价，如 ¥109.00必填
按教室配置容量选择门店和教室，分别设置容量上限可选
媒体模板课程图片、视频、介绍（需提前创建）可选

排课条目列表支持按「课程ID」「课程名称」「课程类型」「状态」筛选搜索。每条条目可执行「查看」「编辑」「删除」操作。

提示：排课条目是后续排课的基础，务必确认类目关联正确，否则会导致小程序端无法筛选到该课程。
图 5-2：排课条目管理 — 课程基础属性配置
必选

Step 4：新建媒体模板

进入 课程管理 → 媒体模板管理，点击「新建媒体模板」：

  1. 选择所属类目（Step 2 中创建的类目）
  2. 上传课程图片（列表封面、详情页头图）
  3. 上传课程视频（可选，用于详情页展示）
  4. 填写课程介绍（支持图文混排，详细介绍课程内容、适合人群、注意事项）
  5. 设置是否为「默认」模板（同一类目下可设置一个默认模板）

媒体模板列表采用分组折叠方式展示，按「排课条目/媒体名称」分组，每组下展示该条目关联的全部媒体。支持「全部展开」和「全部收起」快速切换视图。列表支持按「模板ID」「模板名称」「状态」筛选搜索。

图 5-3：媒体模板管理 — 课程内容素材配置（分组折叠展示）

Step 5：启用排课条目

在排课条目列表页，找到该课程，点击「启用」。状态变为「启用」后，该课程即可用于排课。

如需在小程序端展示给用户浏览，还需完成后续排课和审批步骤。

完成检查：启用后务必确认：1）类目已启用；2）小程序分类已启用；3）排课条目已启用；4）关联的媒体模板状态正常。

📅 如何进行排课

适用于：每周固定排课、临时加课、节假日调课等场景。创建后需审批通过才可在前端展示。

📍 操作路径：课程管理 → 团体课 → 排课管理 → 列表视图 → 新建排课

排课前准备

排课前请确认以下信息已配置完成：

  • ✅ 排课条目已创建且状态为「启用」（见「如何新建一门课程」）
  • ✅ 目标门店和教室已配置
  • ✅ 授课教练已录入系统且状态为「在职」
  • ✅ 教练已在「日程报备」中标记该时段为「可排课」

排课操作步骤

1

选择排课条目

在排课管理页面点击「新建排课」，选择已启用的排课条目。系统自动带入容量和价格。

2

设置时间与地点

选择上课日期、开始时间、结束时间；选择门店和教室。

3

分配教练

从教练列表中选择授课教练。系统会自动检查教练该时段是否有冲突（已有排课或已报备不可用）。

4

确认并创建

确认信息无误后点击「确认创建」，课表默认状态为「草稿」。

5

提交审批并发布

点击「提交团课排课发布审批」，审批通过后课程即时在小程序端展示，用户可进行浏览和预约。

🔒 排课发布审批流
运营提交 → 运营主管审批 → 审批通过后前端展示
注意事项：已有人预约的课程不可随意取消或修改时间，如需调整需提前通知已预约用户并协助改签。审批通过后的排课才会对用户可见。

🏢 如何新增一家门店

适用于：新门店开业前，在系统中完成基础信息录入。

📍 操作路径：门店管理 → 门店列表 → 新增门店

Step 1：填写门店基础信息

点击「新增门店」，填写以下信息：

字段填写说明
门店编码唯一标识，建议按「城市缩写-区域-序号」规则，如 BJS-CHAOY-031
门店名称品牌门店名称，如「北京视频测试门店」
城市/区域所属城市与行政区
详细地址完整地址，用于导航和配送
联系电话 / 营业时间门店客服电话与每日营业时间
容量门店最大容纳人数
评分初始评分，开业前可设为系统默认值
展示状态营业中 / 装修中 / 已关闭（新门店先选「装修中」）
门店负责人指定门店负责人，用于接收投诉报修工单
图 6-1：门店列表 — 全部门店信息一览

Step 2：发布门店公告（可选）

在门店详情页可发布临时公告（如闭店通知、装修通知、活动预告），公告会同步到用户端小程序门店详情页。

Step 3：配置门店关联信息

门店创建完成后，还需要配置以下内容（详见后续章节）：

  • 🏠 教室：为该门店添加教室（见「如何配置教室」）
  • 🚪 门禁：为门店各出入口配置门禁设备（见「如何管理门禁设备」）
  • 👨‍🏫 教练：将门店关联的教练设置所属城市区域

Step 4：处理投诉报修

用户或客服提交的门店相关投诉/报修，会生成工单流转至门店负责人：

  1. 工单自动分配给门店负责人
  2. 负责人处理后可更新工单状态并填写处理结果
  3. 处理进度实时同步给提交人（客服/用户）
开业流程：装修中 → 完成教室/门禁/教练配置 → 上传课程排期 → 将门店状态改为「营业中」→ 小程序端即可预约。

🏠 如何配置教室

适用于：新门店开业、现有门店改造新增教室时。

📍 操作路径：门店管理 → 教室管理 → 新建教室

新建教室步骤

1

选择所属门店

从下拉菜单中选择教室归属的门店。

2

填写教室名称

如「团课教室A」「私教室1」，限 20 字以内，建议统一命名规范。

3

设置容纳人数与面积

容纳人数决定课程最大容量上限；面积用于门店信息展示。

4

选择教室类型

团课教室 / 私教室 / 功能室等，不同类型影响可选的课程类型。

5

配置设施设备

勾选教室内已有的设施（音响系统、镜面墙、空调、LED屏、地板、瑜伽垫、动感单车、器械设备、哑铃架、壶铃、弹力带、拉伸区等）。如预设选项不满足，可在底部「输入自定义设施名称，按回车添加」中自行补充。

6

填写教室描述

选填，0-300字，用于介绍教室特色或适用课程类型，帮助用户了解教室环境。

7

上传教室图片

必填，上传教室实拍图，用于小程序端门店详情展示。建议上传高清实景图，提升用户预约意愿。

图 7-1：新建教室 — 教室信息录入表单（含自定义设施与描述）
提示：教室创建后可在列表中编辑。删除教室前需确认该教室无未来排课记录，否则会导致预约数据异常。

🚪 如何管理门禁设备

适用于：新门店安装门禁、设备故障更换、远程开门等场景。

📍 操作路径：门店管理 → 门禁管理

新增门禁设备

点击「新增门禁设备」，填写以下信息：

  • 设备名称：便于识别的名称，如「深圳测试教室17号门」
  • 设备序列号：硬件出厂编号，务必准确录入
  • 门禁类型：闸机 / 门 / 教室门禁
  • 门店 / 教室：设备安装的物理位置
  • 机构ID：设备归属的机构标识
  • 业务状态：草稿（调试中）/ 启用（正式运行）
图 8-1：门禁设备管理列表

门禁高级配置

在门禁管理页面，除了设备列表外，还提供以下高级配置 Tab：

  • 策略配置：设置门禁通行策略，如预约用户通行时段、会籍卡用户通行权限等
  • 用户策略：针对特定用户或用户组的通行权限管理

日常操作

操作场景说明
查看查看设备详情和配置信息查看设备完整信息，包括关联门店/教室
开门用户忘带手机/人脸识别失败远程控制门禁开启，需记录原因
日志排查异常通行记录查看该设备的全部通行记录
编辑设备更换门店/教室修改设备位置信息
删除设备报废删除后该设备不再同步用户人脸数据
重要提醒：「开门」操作会直接影响门店物理安全，执行前需核实请求人身份，非紧急情况建议由门店前台处理。

🎯 如何添加新教练

适用于：新教练入职时，在系统中录入档案。

📍 操作路径：教练管理 → 教练列表 → 添加教练

录入教练信息

点击「添加教练」，填写以下信息：

字段填写说明
昵称对外展示的名称，如「HH5教练」
真实姓名真实姓名，用于合同与薪酬发放
手机号联系方式，用于系统通知
性别男 / 女
经验年限从业年限，如「4年经验」
角色/等级主教练 / 助教 + Level 等级（Level 1~6）
合作方式全日制 / 兼职 / 实习
城市区域所属门店的城市与行政区
雇佣状态在职 / 离职

教练列表支持按「昵称」「真实姓名」「手机号」「角色」筛选搜索。每条记录可执行「查看」「编辑」「删除」操作。

图 9-1：教练列表 — 全量教练档案

教练状态管理

在教练列表页可直接管理教练状态：

  • 启用 / 停用：通过列表中的状态开关快速启用或停用教练账号，停用后该教练不可被排课
  • 雇佣状态：可直接在列表页切换教练的雇佣状态（在职 / 离职），离职后系统保留历史数据但不再允许排课

教练入职后需完成的配置

教练档案创建后，还需完成以下配置才能正常排课：

  • ✅ 配置薪酬：设置课时费标准（见「如何配置教练薪酬」）
  • ✅ 录入人脸：教练到店完成人脸识别录入，用于门禁通行
  • ✅ 提交日程报备：教练在小程序端提交可授课时段

💰 如何配置教练薪酬

适用于：新教练入职定薪、老教练调薪、月度薪酬核算时。

📍 操作路径：教练管理 → 教练薪酬 / 教练绩效管理 → 课时费配置

Step 1：进入教练绩效管理页

在「教练绩效管理」页面，顶部分析卡展示整体数据：

  • 教练总数：全部在职教练人数
  • 已配置 / 待配置：已设置课时费的教练数量
  • 本月总收入 / 总课时 / 人均收入：用于薪酬预算参考

页面提供以下子 Tab，可按需切换查看：

  • 课时费配置：查看和编辑所有教练的课时费标准
  • 绩效记录：查看教练历史绩效数据和变动记录
  • 数据分析：按时间维度分析教练产能、收入趋势和流失率
  • 配置模板：创建薪酬模板，便于批量应用
图 10-1：教练绩效管理 — 课时费与收入分析

Step 2：配置课时费

找到目标教练，点击「编辑」进行课时费配置：

字段说明
团体课时费团体课每节课的基础报酬
私教课时费私教课每节课的基础报酬
训练营课时费训练营每节课的基础报酬
提成比例超出基础课时的提成百分比

每条薪酬记录可执行「编辑」「复制」「记录（查看操作日志）」「删除」操作。点击「复制」可快速复制当前教练的薪酬配置给新入职教练。

Step 3：设置类目特殊规则（可选）

对于部分课程类目，可设置差异化薪酬规则：

  • 例如：瑜伽课基础课时费 ¥0.01/节，保底金额 ¥150.00/节
  • 综合体能训练基础课时费 ¥200.00/节，课时费比例 90%
  • 设置生效开始日期和生效状态，支持长期有效或限时生效
图 10-2：教练薪酬详情 — 类目特殊规则配置（含保底金额与生效状态）

其他薪酬相关页面

在教练薪酬模块顶部还设有以下 Tab：

  • 课程类型课时费：按课程类型统一设置课时费标准，适用于批量调整同类课程的教练报酬
  • 教练薪酬报表：生成教练薪酬汇总报表，支持按月份、门店、教练筛选并导出 Excel，用于财务对账
批量操作：对于薪酬标准相同的教练，可先在「配置模板」中创建模板，再使用「批量配置」一键应用。

📋 如何处理教练日程报备

适用于：教练请假、培训、休假时，标记不可排课时段。

📍 操作路径：教练管理 → 日程报备

查看教练报备记录

在「日程报备」页面可查看所有教练提交的报备记录。支持按「教练」（下拉选择）和「报备时间范围」（开始日期-结束日期）筛选搜索。

字段说明
主单ID报备记录的唯一标识编号
教练报备教练姓名与ID
排课状态可排课（绿色）/ 不可排课（红色）
报备时段不可排课的具体时间段，可展开查看全部时段
备注报备原因：请假 / 培训 / 休假等
提交时间报备提交的时间
提示：排课系统会自动读取报备数据，在「不可排课」的时段内无法为该教练安排课程。每条报备记录可点击「查看」查看详情。
图 11-1：日程报备 — 教练不可用时段管理

处理流程

1

收到报备

教练在小程序端提交日程报备，后台自动同步至「日程报备」列表。

2

核对排课冲突

检查该教练在报备时段内是否已有排课。如有排课，需联系客户协助改签或取消。

3

确认报备

后台无需手动审批，报备自动生效。如需撤销（如教练临时取消请假），联系教练在小程序端撤销报备。

注意：教练报备「不可排课」后，系统会自动阻止该时段的排课操作，但已有人预约的历史排课不受影响，需人工处理。

💎 如何创建储值卡

适用于：推出新储值卡产品、调整现有卡面或价格时。

📍 操作路径：卡券管理 → 储值卡 → 快速创建储值卡

Step 1：创建储值卡产品

点击「快速创建储值卡」，配置以下信息：

  • 储值卡名称：如「VIP储值卡3000元」「新人专享储值卡1000元」
  • 售价：用户实际支付金额
  • 赠送金额：购买时额外赠送的金额（如充 1000 送 100）
  • 有效期：如 365 天，从购买日起计算
  • 卡面设计：上传储值卡封面图，建议按品牌 VI 统一风格

顶部数据卡片实时展示：储值卡总数（已启用张数）、售卖总额（含订单笔数）、赠送金额（含赠送次数）、兑换码（含使用率）、用户沉淀金额（含沉淀用户数）。

图 12-1：储值卡管理系统 — 卡片列表与数据概览

Step 2：上架与售卖监控

创建后卡片默认状态为「启用」，自动在小程序端展示。储值卡管理页面提供以下 Tab：

  • 储值卡管理：查看卡片列表，展示已售数量、营收数据，支持停用/编辑/删除
  • 售卖管理：查看每笔储值卡购买的订单详情，含购买用户、数量、单价、总金额、支付方式、售卖时间、操作员；支持按时间筛选、搜索订单号/用户、导出 Excel
  • 赠送体系：管理储值卡赠送活动规则
  • 兑换码系统：生成和管理储值卡兑换码，监控兑换率和使用数据
  • 数据中心：汇总储值卡整体运营数据，支持多维度分析
图 12-2：储值卡售卖管理 — 售卖记录与统计分析
营销策略：新用户首次购买建议设置「新人专享」标签和较高赠送比例，提升首单转化率；高面值卡面向忠实会员，用于提升客单价。

🎟️ 如何创建次卡产品

适用于：推出按次计费的课程包产品，如「10次瑜伽课包」。

📍 操作路径：卡券管理 → 次卡模板 → 次卡售卖包

Step 1：创建次卡模板

次卡模板定义了次卡产品的通用规则，创建后可供多个售卖包复用：

  1. 进入 卡券管理 → 次卡模板，点击「新建」
  2. 设置转赠规则：是否允许转赠、是否允许赠课、转赠价格说明
  3. 设置使用金额规则：单课程最大抵扣金额
  4. 上传卡面图片：建议统一品牌风格
  5. 提交「次卡模板发布审批」，通过后模板可用
🔒 次卡模板发布审批流
运营提交 → 运营主管审批 → 模板可用
图 13-1：次卡模板详情 — 转赠规则与卡面配置

Step 2：创建次卡售卖包

次卡售卖包是将模板打包为可售卖的商品，用户实际购买的是「售卖包」：

字段说明
售卖包名称商品名称，如「首次使用后激活」
售卖包类型自由组合包 / 固定包
关联模板选择 Step 1 中创建的次卡模板
售卖价用户实际支付价格
原价划线原价，用于展示优惠力度
总次数该卡包含的总课次数
已售/库存固定包可设置库存上限；自由组合包不限库存

如需售卖，还需配置价格和库存，提交「次卡售卖审批」。

次卡售卖包列表支持按「售卖包名称」「关联模板」「状态」「售卖状态」筛选搜索。每条记录可执行「查看」「编辑」「下架」操作，下架后前端不再展示该售卖包。

图 13-2：次卡售卖包 — 商品组合与定价配置

Step 3：次卡发放策略

在「次卡发放策略」Tab 中，可查看和管理所有次卡的发放记录：

字段说明
发放类型用户ID / 批量发放等
发放方式立即发放 / 定时发放
目标数量计划发放的数量
生效日期次卡生效的开始日期
创建人 / 创建时间记录发放操作人和操作时间

次卡延期与冻结

在用户卡券详情页，支持对次卡进行延期、冻结/解冻操作：

  1. 找到目标用户的次卡资产
  2. 点击「延期」可延长有效期（需填写原因）
  3. 点击「冻结/解冻」可暂停或恢复次卡使用（需填写原因）
  4. 所有操作均留操作日志，便于审计
类型说明：「自由组合包」允许用户按需选择课程组合；「固定包」为预设课程组合，适合标准化产品销售。

🏷️ 如何创建优惠券

适用于：节日促销、新客拉新、老客召回等营销活动。

📍 操作路径：卡券管理 → 优惠券模板 → 新建

创建优惠券步骤

1

设置基础信息

进入 卡券管理 → 优惠券模板，点击「新建」。填写优惠券名称、选择优惠券类型（单品优惠券 / 满减券 / 折扣券）、设置优惠方式（满减 / 直减 / 折扣比例）。

2

配置优惠金额

如「满100减10」「直减50元」。设置是否允许「赠送好友可用」。

3

设置适用范围

限制优惠券可用的城市、门店、教室、排课。如「仅限北京门店」「仅限瑜伽类课程」。

4

提交审批并启用

提交「优惠券模板发布审批」，通过后状态变为「已启用」，可通过运营活动、短信、小程序弹窗等渠道发放给用户。

🔒 优惠券模板发布审批流
运营提交 → 运营主管审批 → 模板可用
图 14-1：优惠券模板详情 — 优惠规则与适用范围

优惠券详情字段说明

在优惠券模板详情弹窗中，除基础信息和适用范围外，还包含以下字段：

  • 优惠券ID：系统生成的唯一标识，可复制使用
  • 说明：优惠券的补充说明文字，可在创建时填写
  • 适用排课：精确到具体排课条目的适用范围限制
  • 操作记录：显示该模板的创建人和更新人，便于追溯责任
营销建议：新客首单建议设置「无门槛直减券」降低决策门槛；老客召回建议设置「满减券」提升客单价；裂变活动建议开启「赠送好友可用」。

适用于：会员招募活动、企业团购、老客户升级等批量发放场景。

创建会籍卡模板

进入 卡券管理 → 会籍卡模板，点击「新建」：

  1. 填写有效期（月卡 / 季卡 / 年卡）、价格
  2. 设置无限次预约权限等权益内容
  3. 提交「会籍卡模板发布审批」，通过后上架

手动发放卡券

在 卡券管理 → 发放记录 中，选择「手动发放」：

  2. 选择卡券模板（会籍卡 / 次卡 / 优惠券）

适用于：在小程序首页、课表页等位置投放广告或活动入口。

📍 操作路径：运营位管理 → 新增运营位 → 启用 / 提交审批

Step 1：创建运营位

点击「新增运营位」，填写以下信息：

字段填写说明
运营位ID系统生成的唯一标识，用于接口对接和排查问题
运营位名称内部管理名称，如「618大促Banner」

投放城市 / 教室选择投放范围，支持按城市或具体教室定向

  • 跳转链接：目标页面 URL

📄 如何更新用户协议

适用于：隐私政策调整、服务条款更新、新增业务需要补充协议时。

📍 操作路径：运营管理 → 协议管理 → 上传新版本

更新协议步骤

1

选择要更新的协议

在协议列表中找到目标协议，如「隐私政策」「自动续费协议」「人脸隐私协议」等。

2

上传新版本

点击「上传新版本」，上传更新后的协议文件（PDF 或 Word）。系统自动生成新版本号（如 v2026052101）。

3

启用新版本

上传后新版本的默认状态为「停用」，点击「启用」后，新用户在注册时将看到最新版本协议。

4

旧版本处理

旧版本自动变为「停用」状态，但历史同意记录仍保留，便于审计追溯。

图 17-1：协议管理 — 用户协议版本维护

协议列表字段与操作

协议列表中展示以下字段：

字段说明
协议名称如「隐私政策」「自动续费协议」
协议编码系统生成的唯一标识，如 PRIVACY_POLICY

法律提示：协议更新涉及用户权益，建议更新前由法务审核，并在小程序中以弹窗形式提示已注册用户重新阅读并同意。

查看教练课时费

⚙️ 如何配置账号权限与操作日志

适用于：新员工入职开账号、权限调整、安全审计。

📍 操作路径：系统管理 → 账号管理 / 操作日志

账号与角色管理

进入 系统管理 → 账号管理：

  1. 点击「新建账号」，填写姓名、手机号、登录邮箱/用户名
  2. 选择角色：系统管理员 / 运营 / 客服 / 财务 / 门店负责人等
  3. 分配菜单权限：控制该账号能看到哪些模块
  4. 分配操作权限：控制该账号能执行哪些操作（如普通运营无退款权限，客服主管可处理退款等）
  5. 保存后账号即时生效，用户可使用手机号+验证码登录
权限设计原则：最小权限原则——只授予完成工作所必需的权限。涉及资金的操作（退款、卡券发放、薪酬修改）建议仅开放给主管级账号。

查看操作日志

所有重要操作均记录操作日志，用于数据安全审计：

记录类型说明
退款操作退款金额、原因、审批人、执行时间
卡券发放发放对象、卡券类型、数量、操作人
教练信息修改修改字段、修改前/后值、操作人
报表导出导出时间、导出内容、操作人 IP
登录记录登录时间、IP 地址、登录设备

支持按时间、操作人、操作类型筛选查询。操作日志不可删除或修改，确保审计可追溯。

安全提醒：发现异常操作（如非工作时间大量退款、陌生 IP 登录）时，请立即冻结相关账号并联系技术负责人排查。
